# 共享元素动画复盘

> 最后更新：2026-05

这份文档记录播放页封面共享元素动画的排查结论。以后新增或修改类似动画前，先按这里的状态模型和检查清单过一遍，避免重复踩同一类坑。

## 这次为什么会出这么多问题

这个功能表面上只是「封面从 A 矩形飞到 B 矩形」，但实际 A/B 都不是稳定坐标。

迷你播放栏封面的 `boundsInRoot()` 会被很多状态影响：

- 搜索时键盘弹出，迷你播放栏被 IME 顶高；
- 首页正在离场，迷你播放栏跟着导航动画移动；
- 系统 inset 正在变化；
- 播放页盖在首页上方，背景里的迷你播放栏仍可能重新布局；
- 共享 overlay、原封面、页面下滑动画同时存在。

最初实现把一个测量值同时当成了几种不同概念：

- 当前帧实时坐标；
- 稳定的迷你播放栏目标坐标；
- 进入动画起点；
- 退出动画终点。

这些语义没有拆开，临时坐标就会污染后续动画。

## 根因

### 实时坐标被当成稳定坐标

`boundsInRoot()` 表示当前这一帧的位置，不代表这个元素的语义目标位置。

搜索输入时，键盘会把迷你播放栏顶到屏幕中段；从首页进入播放页时，首页离场动画也会让迷你播放栏持续移动。这些位置在当帧是正确的，但不能保存成之后关闭播放页时的目标点。

### 导航动画和共享元素动画时序互相影响

播放页下滑、原封面隐藏、迷你播放栏封面显示、共享 overlay 移动，这几件事发生在同一个视觉窗口内。

只要时序差一帧，就会出现：

- 多出一张封面；
- 封面先消失再出现；
- 封面朝旧坐标飞走；
- 非方形封面第一次收起方向明显偏移。

### 状态命名和职责不清

一个 `miniCoverBounds` 不能同时回答这些问题：

- 迷你封面现在在哪？
- 进入播放页时从哪里起飞？
- 退出播放页时飞到哪里？
- 这个坐标来自稳定布局，还是键盘/导航过渡中的临时布局？

必须把这些状态拆开。

## 正确状态模型

以后做共享元素动画时，至少区分这些概念：

- `liveBounds`：当前帧测量值，只用于观察，不长期信任。
- `stableTarget`：只在稳定 UI 状态下更新的目标坐标。
- `transitionFrom`：动画启动瞬间冻结的起点快照。
- `transitionTo`：动画启动瞬间冻结的终点快照。
- `overlayReady`：overlay 图片已准备好，可以安全隐藏原元素。
- `suppressStableUpdates`：进入/离开页面、键盘显示、共享动画期间阻止污染 stable。

## stable 坐标更新规则

以下状态禁止更新稳定目标坐标：

- 键盘/IME 可见；
- 来源页面正在进入或离开；
- 导航路由动画正在进行；
- 共享元素 overlay 正在运行；
- 测量值只是某个过渡帧的位置。

迷你播放栏封面尤其要注意：

- 从首页进入播放页前，先冻结 stable 写入；
- 从搜索结果或列表项进入播放页前，也要冻结 stable 写入；
- 键盘显示时忽略迷你播放栏测量值；
- 关闭播放页时，不盲信某一个 state，而是从当前 mini、stable、fallback 候选里选择最接近真实底部位置的目标；
- fallback 不能作为唯一裁判，因为 inset/IME 过渡时 fallback 本身也可能是临时值。

## overlay 接管规则

用 overlay 替换原封面时，顺序不能颠倒：

1. 在冻结的 `from` 坐标创建 overlay。
2. 等 overlay 图片 ready。
3. 隐藏原封面。
4. 启动页面转场和共享元素移动。
5. 动画结束后清理 overlay 临时状态。

这样可以避免原封面先隐藏、overlay 还没解码完成导致的空帧闪烁。

## 必测场景

类似共享元素动画完成前，至少测试：

- 迷你播放栏正常打开/关闭播放页；
- 从列表歌曲进入播放页；
- 从搜索结果进入播放页；
- 搜索框键盘显示时进入播放页；
- 键盘刚关闭或正在关闭时退出播放页；
- 播放页内切歌后第一次退出；
- 方形封面；
- 竖向非方形封面；
- 横向非方形封面；
- 减少动态效果模式；
- 所有会上报封面坐标的迷你播放栏样式。

## 设计准则

不要把共享元素动画设计成「从测量到的 A 动到测量到的 B」。

要把它设计成一个小状态机：

1. 收集实时测量值；
2. 只在稳定状态接受测量值进入 stable；
3. 动画启动时冻结起点和终点；
4. 等 overlay ready 后再交接显示；
5. 运行动画；
6. 动画结束后清理临时状态。

如果一个坐标会被键盘、inset、路由转场、滚动、背景页布局影响，它默认就不是稳定坐标。
