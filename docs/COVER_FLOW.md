# 播放页封面流特殊主题设计

> 目标：在不破坏当前播放页结构的前提下，让播放页拥有一个常驻、有记忆点的横向封面带表现。

---

## 设计定位

**平行封面带**是播放页的特殊主题：启用后播放页封面区始终使用横向并排的队列封面带，不再按播放 / 暂停状态放大回半屏大封面。

- 播放中：上一张 / 当前 / 下一张以同尺寸、同朝向并排展开，当前专辑图仍是中心主视觉。
- 暂停：保持同一套平行封面带，不再触发布局切换。
- 封面显示：该主题下播放页封面强制使用裁切填充，自动禁用「原样比例」。

一句话：**播放页始终把队列铺成一条平行封面带。**

---

## 状态

### 平行封面带态

触发条件：

- 自动：启用主题且播放页封面区可用时常驻进入。
- 退出：进入歌词聚焦或下半屏沉浸时临时退出，返回普通播放页布局过渡。

视觉参数建议：

| 元素 | 建议 |
|------|------|
| 当前封面 | 缩放到约 `0.76`，保持直角、清晰、不旋转 |
| 左右第一张 | 与当前封面同尺寸、同朝向，横向并排，不与当前封面重叠 |
| 左右第二张 | 继续平行外排，可降低透明度弱化层级 |
| 倒影 / 阴影 | 每张封面下方保留完整镜像图的上部裁剪倒影，显示高度约封面 `28%`，中段开始渐隐，`28%` 以下透明；不做整屏玻璃舞台 |
| 背景 | 复用现有播放页背景，不新增舞台、皮革、重玻璃质感 |

## 交互

- 点击侧边封面：切到对应歌曲，并保持折叠态短暂展示。
- 横向拖动封面区：跟手平移封面带，松手超过约 35% 屏宽切歌；未达阈值回弹。实现见 [`CoverGestureCoordinator.kt`](../app/src/main/java/com/mica/music/ui/screens/player/CoverGestureCoordinator.kt) 与 [Lane 池文档 §4.2.1](COVER_FLOW_LANE_POOL.md#421-滑动切歌跳变根因与正确时序2026-06-已修)。
- 播放 / 暂停按钮：不改变平行封面带状态。
- 减少动态效果开启时：直接切状态，不播放 3D 插入动画。

---

## 动效

所有动画进入 `MicaMotion` 体系。

| 变化 | 建议 |
|------|------|
| 进入平行封面带 | Long `400ms`：当前封面缩小，左右封面从屏幕边缘淡入并排展开 |
| 临时退出平行封面带 | Medium `320ms`：左右封面向外淡出，当前封面回到普通播放页布局 |
| 切歌换位 | Medium `320ms`：封面带整体横向平移，旧当前封面移动到相邻位置，新封面移动到中心 |
| 复古立体切歌 | Long `400ms`：封面沿连续中心索引移动；侧边封面靠近中心时逐步转正，当前封面离开中心时逐步倾斜，避免角度跳变 |
| 减少动态效果 | `0ms`，直接更新布局 |

播放页已有约束仍适用：先冻结布局参数，再计算目标，再由单一 progress 驱动 `lerp` / fade，避免动画过程中反复重算间距。

## 与现有设置的关系

建议新增设置项：

- 「播放页封面行为」：
  - 标准
  - 平行封面带
  - 复古立体封面

该项与「播放页背景」并列但互不替代。也就是说，平行封面带可以同时搭配：

- 主题色
- 封面渐变
- 封面模糊
- 后续可能加入的倒影模糊渐变

启用任一封面流特殊主题时，播放页封面区域局部忽略「封面显示」里的「原样比例」，统一按裁切填充渲染，以保证封面流中的尺寸一致。

### 复古立体封面

**复古立体封面**恢复经典 Cover Flow 的空间感：当前封面保持正面居中，两侧相邻封面向内做透视倾斜，形成老式唱片浏览架的层次。

| 元素 | 建议 |
|------|------|
| 当前封面 | 约 `0.62` 缩放，正面展示，保持中心专辑的主视觉重量 |
| 左右第一张 | 约 `0.52` 缩放，向内 `75°` 透视倾斜，位置约 `0.81` 屏宽 |
| 左右第二张 | 约 `0.44` 缩放，位置约 `0.90` 屏宽，贴近第一侧与屏幕边缘之间的外侧间隙，并保持完全不透明 |
| 倒影 | 复用平行封面带的完整镜像裁剪倒影，每张封面独立跟随透视与位移 |

## 第一版范围

第一版建议只做播放页封面区：

- 使用当前队列中当前曲前后各 1–2 首作为侧边封面。
- 启用后常驻平行封面带，不再支持播放中自动放大回半屏大封面。
- 侧边封面可点击切歌。
- 暂不做专辑浏览页 Cover Flow。
- 暂不做强拟物舞台、全量倒影、皮革或金属主题。

这样可以保留当前 Mica 的克制感，同时给播放页一个明确的新主题记忆点。
